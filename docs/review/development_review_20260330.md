# コードレビュー結果

## レビュー対象

- `apps/webapp/` — Spring Boot 4.0.5 + Java 25 + MyBatis アプリケーション（新規作成、64 ファイル）
- `ops/scripts/webapp.js` — webapp セットアップ・運用スクリプト

## 総合評価

Spring Boot 4 + Java 25 への先進的な移行と、MyBatis Mapper の CRUD テスト網羅は基盤として堅実。ただし、アーキテクチャ設計で定義されたドメインモデルパターンに対して、現状のドメインクラスは getter/setter のみの Anemic Domain Model であり、ビジネスルール（BR01-BR08）がコード上で未実装。テスト戦略の「ユニットテスト 80%」に対してドメイン層のユニットテストがゼロという乖離が最大の課題。

## 改善提案（重要度順）

| # | 提案 | 箇所 | 指摘元 | 理由 |

### 改善提案

- 【高】ドメインモデルが Anemic Domain Model。ビジネスルール BR01-BR08 をドメインモデル内にメソッドとして実装すべき
- 【高】受注の不変条件検証がない。コンストラクタまたはファクトリメソッドで BR01 を強制すべき
- 【中】値オブジェクトが欠落（価格、メールアドレス、電話番号等）
- 【中】Mapper XML で `updated_at` が手動更新されていない
- 【中】webapp.js:53 の `runInNix` シングルクォートエスケープ不足
- 【低】Testcontainers の BOM 指定はあるが統合テストがない
- 【低】OrderStatus enum に遷移ルールが未実装

- ロジック散在の「手続き型スパゲティ」リスク
- パスワードハッシュ化メカニズム未実装
- ID 型が Long で型安全性が低い
